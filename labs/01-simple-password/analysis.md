# Lab 01 - Simple Password Check

## Goal

This lab demonstrates how a simple password check can be identified inside a compiled Windows executable.

The purpose of this lab is not just to write a C program. The real goal is to understand how basic program logic can be recovered from a binary using reverse engineering tools.

In this lab, we analyze:

- user input handling
- hardcoded string comparison
- the role of `strcmp`
- useful strings inside the binary
- the `main` function inside Ghidra
- the decision logic behind access granted and access denied

---

## Source Code Logic

The program asks the user to enter a password:

```c
printf("Enter password: ");
scanf("%31s", input);
```

The user input is stored inside the `input` buffer.

Then the program compares the entered password with the hardcoded password `bora123`:

```c
if (strcmp(input, "bora123") == 0) {
    printf("Access granted\n");
} else {
    printf("Access denied\n");
}
```

The logic is simple:

```text
If the entered password is equal to bora123, print Access granted.
Otherwise, print Access denied.
```

---

## Important Function: strcmp

The `strcmp` function compares two strings.

Example:

```c
strcmp("bora123", "bora123")
```

Since both strings are equal, the function returns:

```text
0
```

That is why the source code checks this condition:

```c
strcmp(input, "bora123") == 0
```

This means:

```text
If the user input is equal to bora123, grant access.
```

If the entered value is different, `strcmp` does not return `0`, and the program executes the `else` branch.

## Useful Binary Clues

In real reverse engineering tasks, the source code is usually not available.

Most of the time, we only have the compiled executable:

```text
simple_password.exe
```

Because of that, readable strings inside the binary are very important clues.

In this lab, the most useful strings are:

```text
Enter password:
Access granted
Access denied
bora123
```

These strings help us understand the purpose of the program.

For example:

- `Enter password:` suggests that the program expects user input.
- `Access granted` suggests a successful condition.
- `Access denied` suggests a failed condition.
- `bora123` is likely the expected password.

## Ghidra Analysis Result

After opening `simple_password.exe` in Ghidra and running auto-analysis, the `main` function shows the following decompiled logic:

```c
int __cdecl main(int _Argc, char **_Argv, char **_Env)
{
    int iVar1;
    char acStack_28 [32];

    __main();
    printf("Enter password: ");
    scanf("%31s", acStack_28);
    iVar1 = strcmp(acStack_28, "bora123");

    if (iVar1 == 0) {
        puts("Access granted");
    }
    else {
        puts("Access denied");
    }

    return 0;
}
```

The variable `acStack_28` is the name generated by Ghidra.

In the original source code, the variable name was:

```c
char input[32];
```

However, variable names are usually not preserved in the final compiled binary. Because of that, Ghidra creates its own local variable names.

In this case:

```text
input ≈ acStack_28
```

---

## Decision Logic

The most important line is:

```c
iVar1 = strcmp(acStack_28, "bora123");
```

This line compares the user input with the hardcoded password `bora123`.

Then the program checks:

```c
if (iVar1 == 0)
```

This means:

```text
If strcmp returns 0, both strings are equal.
```

So if the user enters the correct password, the program executes:

```c
puts("Access granted");
```

If the password is wrong, the program executes:

```c
puts("Access denied");
```

---

## Why printf Became puts

In the original source code, we wrote:

```c
printf("Access granted\n");
```

However, Ghidra may show it as:

```c
puts("Access granted");
```

This is normal.

The compiler can optimize simple `printf` calls into `puts` calls when no formatting is needed.

The logic is still the same:

```text
The program prints the Access granted message.
```

## What We Learned

This lab shows that:

- simple password checks can be understood from a compiled binary
- readable strings are useful reverse engineering clues
- `strcmp` is an important function when analyzing password checks
- Ghidra can recover C-like pseudocode from assembly
- generated variable names such as `acStack_28` may represent original local variables
- source code is not required to understand basic program logic
